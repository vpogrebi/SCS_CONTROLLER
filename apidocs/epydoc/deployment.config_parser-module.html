Module config_parser

source code


Created on Jul 2, 2009

@author: valeriypogrebitskiy

config_parser.py is a a "wrapper" for standard Python's ConfigParser.ConfigParser class.
Its main goal - is to simplify use of ConfigParser and provide a single-line way of 
loading configuration data from config. file

This functionality is implemented by Parser class, and simplified use of it
is exposed to users via single method getParams()

Examples:

        a. "config_parser.getParams(<path/file.cfg>)" - load config. parameters from ALL the sections in <path/file.cfg>
        b. "config_parser.getParams(<path/file.cfg>, 'SECTION1')" - load config. parameters from single section of <path/file.cfg>
        c. "config_parser.getParams(<path/file.cfg>, ('SECTION1', 'SECTION2')" - load config. parameters from multiple sections of <path/file.cfg>

getParams(config_file_or_filelist, section_or_sectionlist=None)

source code 

Obtain configuration parameters from a single file or a list of files - either for all sections, single section, or for a list of sections (<section_or_section_list>)

Parameters:
  • config_file_or_filelist - str or list or tuple - single configuration file (complete path/file), or list (or tuple) of configuration files
  • section_or_sectionlist - str or list or tuple - single section name, or list (or tuple) of section names
Returns:
dict - dictionary {<param_name>: <param_value>} containing configuration data for all the files specified in @param config_file_or_filelist
Raises:
  • RuntimeError - if @param section_or_sectionlist is not str, list or tuple

getConfigParams(*sections)

Read system configuration parameters from config file defined by $CONFIG_FILE environment variable

Parameters:
  • sections - list of configuration sections to load from config file. If None is passed - all sections will be loaded
Returns:
params - configuration parameter dictionary {<param>: <value>}
Raises:
  • RuntimeError - if $CONFIG_FILE environment variable is not defined
