Module d.singleton

Part of deployment 

'singleton.py provides Singleton design pattern implementation that can be used
(when subclassed from singleton.Singleton) to enforce that only single instance
of a class is ever created.

In addition to Singpleton class, this module offers @uniq decorator that can be
used to decorate __init__() methods (constructors) of classes that inherit from
Singleton - to prevent re-initialization of an instance of that class. 

Together, Singleton class and @uniq decorator provide a way to implement a singleton
object that gets initialized only once. Thus, already initialized instance of such 
class can be accessed from anywhere in the code by simply executing 

"obj = ClassName()" 

without providing any arguments. This means that such class can be properly instantiated 
once (by specifying constructor's arguments), and then accessed without providing
arguments to class's constructor by executing "obj = ClassName()"
